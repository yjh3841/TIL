# Today I Learn

MVVM (Model View ViewModel) 패턴

v <-> vm <-> m

view(dom으로 화면 ) -> 가상dom(특정 로직을 수행) -> Model(서버 데이터를 JS객체 저장) -> 데이터바인딩(view에 모델의 데이터 동기화) -> view(사용자에게 보임)

페이스북 읽지 않은 메세지를 MVC로 하다가 복잡해져서 오류가 많이 발생하게 됨 그래서 MVVM 패턴으로 바꿈
